Para atender aos requisitos técnicos do projeto, foi utilizado o laço de repetição while, garantindo que a execução ocorra somente enquanto uma condição previamente definida for verdadeira. A estrutura adotada utiliza laços aninhados: um laço externo é responsável por controlar qual tabuada está sendo gerada (de 1 até 10), enquanto um laço interno executa as multiplicações individuais de cada número, também de 1 até 10. Dessa forma, para cada número definido no laço externo, o laço interno percorre todas as multiplicações correspondentes, produzindo os resultados automaticamente.

O gerenciamento correto do ciclo de repetição é um ponto fundamental da implementação. As variáveis de controle são declaradas e inicializadas antes do início dos laços, garantindo organização e clareza no fluxo do programa. A condição de parada é definida de maneira explícita (limitando a contagem até 10), evitando a ocorrência de loops infinitos. Além disso, os contadores são incrementados ao final de cada iteração, assegurando que o programa avance corretamente até concluir todas as tabuadas.
